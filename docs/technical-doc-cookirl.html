Cook IRL — Technical Overview

A cooking accountability system that takes recipes from wherever you find them, keeps them organized, and pushes you to actually cook them.

Core Features

1. Universal Recipe Import

Import recipes from any source with AI-powered extraction:

SourceHow It Works
URLPaste any recipe link — AI extracts structured data
Video ShareShare cooking videos from TikTok, Instagram, YouTube, Facebook directly to Cook IRL
PDF CookbookUpload a cookbook PDF — AI extracts individual recipes from the entire book
Camera ScanScan a physical cookbook page or recipe card with your camera
Manual EntryAdd your own recipes by hand

AI Pipeline:

Input → LangGraph Router → [PDF|Image|Video|URL|Text] → Gemini 2.5 Flash → Structured Recipe
  • Images: Vision API (OCR + labels) → Gemini parsing
  • Videos: Native multimodal processing (frames + audio in single pass)
  • URLs: Web scraper + structured extraction
  • PDFs: pdf-parse + Vision API for scanned documents

2. Smart Recipe Search

Three ways to find recipes:

ModeHow It Works
Natural LanguageType queries like "quick pasta under 30 mins" — Pinecone vector search understands intent
Voice SearchSpeak your query — Whisper transcribes, AI searches
Ingredient ScanScan items with your camera — find recipes you can make now

Hybrid Search Engine:

// Vector embedding + metadata filters
const results = await pinecone.query({
  vector: embedding,  // text-embedding-3-small (512 dims)
  filter: { userId, categories: { $in: ["dinner"] }, cookTime: { $lte: 1800 } },
  topK: 20
});

3. Cookbook Organization

Create your own cookbooks inside the app to group and organize recipes however you want.

4. Tasks & Reminders

From recipe to reality:

FeatureDescription
Task ListsCreate cooking tasks and grocery shopping lists
Recipe LinkingAttach tasks directly to recipes ("Buy ingredients for Lasagna")
Grocery ListsTap any ingredient to add it to your shopping list
RemindersPush notifications so you don't forget to cook or shop

Server-Side Scheduler:

Scheduler (every 5 min) → Query due tasks → FCM Topic (user_<uid>) → Push notification

Notification types: Shopping | Cooking

5. iOS Home Screen Widgets

WidgetSizesShows
Task WidgetSmall, Medium, LargeCurrent tasks, cooking plans, grocery shopping
Lock Screen WidgetLock screenWhat's in your tasks, cooking, or grocery list

Built with WidgetKit — native iOS performance, background refresh, deep linking into the app.

6. Story Mode (Cooking Interface)

Step-by-step cooking instructions presented Instagram Stories-style. Each step shows what you need, what to do, and timing. Finish a recipe and confetti celebrates your completion.

7. Export & Share

Export any recipe as a PDF and share it with anyone.

// Fetch offerings (OfferingNotifier)
void fetchOfferings() async {
  final Offerings offerings = await Purchases.getOfferings();
  state = state.copyWith(data: offerings.current);
}

// Purchase flow (CurrentPackageNotifier)
Future<void> purchase(bool restore, Function() onCompleted) async {
  if (restore) {
    final info = await Purchases.restorePurchases();
    if (info.activeSubscriptions.isEmpty) {
      showToast("No active purchases found");
    }
    return;
  }

  final purchaseParams = PurchaseParams.package(selectedPackage);
  await Purchases.purchase(purchaseParams);
  onCompleted();
}

Purchase Event Handler:

async handlePurchaseEvent(event, user) {
  const subscription = {
    id: event.transaction_id,
    productId: event.new_product_id ?? event.product_id,
    entitlements: event.entitlement_ids || [],
    promoCode: event.offer_code ?? null,
    durations: {
      startAt: event.purchased_at_ms,
      endAt: event.expiration_at_ms,
    },
    status: "active",
  };

  await userService.updateSubscription(user.id, subscription);
  
  if (event.offer_code) {
    await this.claimPromoCode(event.offer_code);
  }
  
  await this.createTransaction(event, user);
}

Subscription Data Model

// Stored in Firestore users/{uid}.activeSubscription
activeSubscription: {
  id: string,                    // transaction_id
  productId: string,             // e.g. "cookirl_pro_weekly"
  entitlements: string[],        // ["pro"]
  promoCode: string | null,      // offer code if used
  status: "active" | "cancelled" | "expired",
  durations: { 
    startAt: number,             // purchased_at_ms
    endAt: number                // expiration_at_ms
  },
  updatedAt: number
}
