This Privacy Policy explains how Paul Jeremiah ("Developer", "we", "us", or "our") collects, uses, discloses, and protects your personal information when you use the Cook IRL mobile application ("App").

By using the App, you consent to the data practices described in this policy.

Contact: hello.paulvin@gmail.com

2. Information We Collect

2.1 Information You Provide

Account Information:

  • Name

  • Profile picture (if provided through Apple/Google Sign-In)

User Content:

  • Recipes you create or import
  • Cookbook collections
  • Recipe images and photos
  • Cooking tasks and reminders
  • Grocery lists
  • Notes and preferences

2.2 Information Collected Automatically

Device Information:

  • Device type and model
  • Operating system version
  • Unique device identifiers
  • App version

Usage Information:

  • Features used and interactions
  • Time spent in the App
  • Crash reports and error logs

Camera and Microphone Data:

  • Photos taken for ingredient scanning (processed locally or uploaded for analysis)
  • Voice recordings for voice search (processed for recipe search functionality)

Note: Camera and microphone access is only used when you explicitly use these features.

2.3 Information from Third Parties

Authentication Providers:

  • Apple Sign-In: Name, email (as you choose to share)
  • Google Sign-In: Name, email, profile picture

Recipe Import:

  • When you import recipes from URLs, we may process publicly available recipe data from those sources

3. How We Use Your Information

We use your information to:

Provide and Improve the Service:

  • Create and manage your account
  • Store and sync your recipes, cookbooks, and preferences
  • Enable recipe import and extraction features
  • Power voice and image search functionality
  • Send cooking reminders and notifications
  • Generate grocery lists

Analytics and Improvements:

  • Understand how users interact with the App
  • Identify and fix bugs and crashes
  • Improve features and user experience
  • Develop new features

Communications:

  • Send important service updates
  • Respond to your inquiries and support requests

Subscription Management:

  • Process payments through App Store/Google Play
  • Manage your subscription status

4. Data Storage and Security

4.1 Where We Store Data

Your data is stored using:

  • Firebase Firestore: Cloud database for user data, recipes, and cookbooks
  • Firebase Storage: Recipe images and media files
  • Local Device Storage: Cached data and preferences

Data is stored on servers located in the United States and other regions where Google Cloud (Firebase) operates.

4.2 Security Measures

We implement industry-standard security measures including:

  • Encrypted data transmission (HTTPS/TLS)
  • Secure authentication through Apple/Google
  • Firebase security rules restricting data access
  • Regular security updates

However, no method of transmission or storage is 100% secure. We cannot guarantee absolute security.

6.1 Access and Export

You can:

6.2 Correction

You can update your:

  • Preferences and settings

You can:

7. Camera, Microphone, and Photo Access

7.1 Camera Access

Used for:

  • Scanning ingredient labels and cookbook pages

Camera access is only activated when you explicitly use scanning or photo features.

7.2 Microphone Access

Used for:

Voice recordings are processed for search functionality and are not stored permanently.

7.3 Photo Library Access

Used for:
